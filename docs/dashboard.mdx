# Dashboard

Once the daemon is running, the dashboard is at
[`http://localhost:30000/dashboard`](http://localhost:30000/dashboard).

## What's on it

- **Dashboard (landing)** — cost/successful task (median), time/successful
  task (median), cost by model, and 8-week trends for memory and task
  throughput.
- **Projects** — every registered project with status, agent count, and
  session count. Drill in for agents and sessions per project.
- **Reviews** — cross-project review groups sorted by most recent.
- **Memories** — approved + applied learnings across projects.
- **Models** — flat model catalog scoped to configured providers, with lab
  filtering and provider details.
- **Session detail** — full conversation logs and Mermaid sequence
  diagrams for each review.

## Main views

### Project summary

The project summary shows review status, active agents, recent sessions,
and the project-specific review loop.

### Agents

The agents view shows the generated specialist fleet for the project.
Recent updates also make it easier to inspect model choices from the
project page:

- the rematch flow is lab-oriented instead of provider-tab oriented
- the best-fit lab picker is scoped to providers already configured for
  that project
- agent names open the detail modal directly
- model detail links jump into the catalog page with the matching model
  preselected

If a project has no configured providers yet, the rematch flow shows an
empty state that points you back to provider setup instead of offering
labs that ATX cannot actually route to.

### Reviews

The reviews view shows review groups, outcomes, follow-up state, and the
tasks attached to each review block. Iterative `request_review` runs now
group into a single active review block instead of scattering across many
one-task rows.

Use this view when you want to answer:

- what is still active
- which blockers were found across iterations
- whether a review loop is converging

### Memories

The memories view shows approved learnings that ATX can reuse in later
reviews.

### Models

The models view is now a flat catalog instead of provider tabs.

- filter by upstream lab with the `All labs` dropdown
- inspect one row per provider-carried model
- the catalog is scoped to providers you have actually configured in ATX
- open the row to see context window, pricing, thinking support,
  sampling defaults, and other metadata
- use it as the source of truth when you want to understand what ATX can
  route to

The current page is useful for both operators and reviewers: it shows the
same catalog ATX uses for routing, including provider-specific coverage of
the same upstream family. If you have zero configured providers, the page
shows an empty-state CTA instead of a misleading global catalog.
